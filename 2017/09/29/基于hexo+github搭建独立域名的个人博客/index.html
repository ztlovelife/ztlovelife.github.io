基于hexo+github搭建独立域名的个人博客

发表于 2017-09-29 | 分类于 计算机黑科技

终于折腾出了自己的博客小站，给大家分享一下教程。如题所说，我的博客是基于hexo+github搭建的，选择这样的搭建方法有以下优点：

  1. 空间免费，无需购买高价钱服务器
  2. 操作简单，相对于普通的搭建网站流程
  3. 完全托管，你只负责更新博客，而不需要维护、备案、升级

当然也有以下缺点：

  1. 空间小，只能做个人博客等小型网站
  2. 速度慢，由于没有数据库支持，打开速度确实很慢
  3. 可能移动端支持度不高，一些页面无法显示

进入正题，先开始准备工作

Github相关操作

github是全球最大的同性程序员交流网站，你可以在这里管理代码、分享项目、围观大神、找好基友

  1. 注册Github，相信大家都会
  2. 创建一个new repository
  3. 仓库的名字取为 yourname.github.io yourname就是你的注册名

Git相关操作

以后都要和这个git打交道，用其中的git bash来进行上传、调试、安装环境等操作，快起官网下载吧

Node.js相关操作

用来生成静态页面的，不可缺少，下载就对了

域名相关操作

  1. 购买域名，这是博客的地址，就像我的www.zhangtong1994.com，阿里云，腾讯云都可以购买，也不贵，几块钱几十块钱就可以买到，大家挑一个喜欢的，网页地址我就不贴了
  2. 购买成功后，进入域名管理界面，点击域名解析，点添加记录，记录类型为CNAME，记录值为你的仓库名字yourname.github.io
  3. 实名认证，这个按流程一步一步就行了，需要提交身份证的照片

至此，准备工作已经结束，准备好撸起袖子大干一场吧

安装hexo

hexo是我们的博客的主体系统，这不是一个准备工作，因为hexo并不是一个软件

  1. 创建一个你管理网站的文件夹，取名随意，我的就叫hexo，以下也都用hexo文件代替
  2. 右键点git bash，弹出神秘代码框，不要惊慌,依次输入
    npm install hexo-cli -g
    npm install hexo --save
  3. 在输入下面代码，如安装成功，可以看到hexo版本号
    hexo -v
  4. 最后再运行,结束安装
    hexo init
  5. 查看本地运行效果
    hexo g
    hexo s

登录本地服务器localhost:4000查看网页效果，这就是我们博客的雏形

对github进行配置

有了本地服务器，我们只要把本地服务器和github的云空间绑定就可以大功告成了

  1. 对hexo文件进行bit bash
    git config --system user.name "yourusername"
    git config --system user.email "youremail@name.com"
  2. 在hexo文件中找到_config.yml文件中，在其中找到deployment，然后配置如下：
    deploy:
    　type: git
    　repo: git@github.com:githubname/githubname.github.io.git
    　branch: master

再运行npm install hexo-deployer-git --save

生成SSH keys

对hexo文件进行bit bash
ssh-keygen -t rsa -C "yourname@name.com"

在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入，我们按回车不设置密码。

然后添加 SSH Key到 GitHub,过程是打开C:\Users\bxm09\.ssh\id_rsa.pub，此文件里面内容为刚才生成的密钥，准确的复制这个文件的内容，粘贴到https://github.com/settings/ssh的 new SSH key 中

将本地文件上传到github服务器

对hexo文件进行bit bash
hexo clean
hexo g
hexo d
然后登陆github，进入你的yourname.github.io这个仓库，添加一个新文件名字叫做CNAME，内容为你的域名（没有https和/），再点击setting，在GitHub Pages这一页找到Custom domain将你的域名填写进去，save一下

至此，我们的博客已经搭建成功，别人可以登录我们的域名看博客了
